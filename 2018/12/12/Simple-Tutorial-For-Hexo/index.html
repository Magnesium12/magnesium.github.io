Tutorial For Hexo

2018-12-12 20:47:25 | Hexo

Hexo 是一款快速,简洁且高效的博客框架, 她具有以下特性

  • 超快速度
  • 支持markdown
  • 一键部署
  • 丰富的插件

安装

在安装前，你必须检查电脑中是否已安装下列应用程序：

  • Node.js
  • Git

如果你的电脑中已经安装上述必备程序，那么恭喜你！接下来只需要使用 npm 即可完成 Hexo 的安装。

Windows 用户

如果你的电脑中尚未安装所需要的程序，请到官网下载Node.js & Git的安装程序进行安装.
注意安装Node.js时注意勾选Add to PATH选项, 这将使得Node.js添加到你的计算机环境变量中.

下载Git可能会有些慢, 请耐心等待或者自行寻找过其他源.

配置

新建完成后, 指定文件夹内目录如下

_config.yml

网站的配置信息, 你可以在这里选择配置大部分参数, 我们称为站点配置文件.

  • 可以设定主题, 网站标题, 副标题, author etc.

package.json

应用程序的信息.

scaffolds

模板文件夹, Hexo会根据scaffolds来建立文件. Hexo的模板是指在新建的markdown文件中默认填充的内容. 例如, 如果您修改scaffold/post.md中的Front-matter内容, 那么每次新建一篇文章时都会包含这个修改.

source

资源文件夹是存放用户资源的地方. 除_posts 文件夹之外, 开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略. Markdown 和 HTML 文件会被解析并放到 public 文件夹, 而其他文件会被拷贝过去.

  • 发布的博文存储在/public/

theme

主题 文件夹。Hexo 会根据主题来生成静态页面。

  • 选择一个你喜欢的主题, it does matter.
  • 主题内会有一个用来配置主题细节的_config.yml文件, 我们称为主题配置文件

常用指令

init

$ hexo init [folder]
新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。

new

`$ hexo new [layout] <br>新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。

generate

`$ hexo generate
生成静态文件。

deploy

`$ hexo deploy
-d, –deploy 文件生成后立即部署网站

$ hexo g
publish
$ hexo publish [layout]
发表草稿。

server
$ hexo server
启动服务器。默认情况下，访问网址为： http://localhost:4000/。

deploy
$ hexo deploy
部署网站。

-g, –generate 部署之前预先生成静态文件
该命令可以简写为：

$ hexo d
render
$ hexo render [file2] …
渲染文件。

clean
$ hexo clean
清除缓存文件 (db.json) 和已生成的静态文件 (public)。

在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。
